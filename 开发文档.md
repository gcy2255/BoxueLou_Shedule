# 博学楼空闲教室查看系统 - 开发文档

## 项目概述

本系统是一个基于前后端分离架构的空闲教室查看系统，用于实时展示博学楼各教室的使用状态。系统通过解析Excel课表数据，结合当前时间，以可视化方式展示教室的空闲/占用状态。

### 技术栈

| 层级 | 技术 | 版本 |
|------|------|------|
| 前端 | Vue 3 + Vite | Vue 3.4, Vite 5.2 |
| 后端 | Spring Boot | 3.2.5 |
| 数据库 | MySQL | 8.x |
| ORM | MyBatis-Plus | 3.5.5 |
| Excel解析 | Apache POI | 5.2.5 |

### 核心功能

1. **教室状态展示**：大屏展示所有教室，绿色表示空闲，红色表示占用
2. **教室搜索**：支持按教室名称模糊搜索
3. **课表详情**：点击教室可查看完整周课表
4. **实时更新**：每分钟自动刷新教室状态

## 环境准备

### 1. 安装必要软件

| 软件 | 版本要求 | 说明 |
|------|----------|------|
| JDK | 17+ | Java开发环境 |
| Maven | 3.8+ | 项目构建工具 |

| MySQL | 8.0+ | 数据库 |

### 2. 配置MySQL

```sql
-- 创建数据库（可选，应用会自动创建）
CREATE DATABASE IF NOT EXISTS classroom_db DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

### 3. 修改数据库配置

编辑 `backend/src/main/resources/application.yml`：

```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/classroom_db?useUnicode=true&characterEncoding=utf-8&serverTimezone=Asia/Shanghai
    username: root      # 修改为你的用户名
    password: root      # 修改为你的密码
```

## 启动项目

### 1. 启动后端

```bash
cd backend

# 首次运行，下载依赖
mvn clean install

# 启动应用
mvn spring-boot:run
```

后端启动后会自动：
- 创建数据库表
- 解析所有Excel文件并导入数据库
- 启动HTTP服务（端口8080）

### 2. 启动前端

```bash
cd frontend

# 安装依赖
npm install

# 启动开发服务器
npm run dev
```

前端启动后访问：http://localhost:5173

### 3. 构建生产版本

```bash
# 构建前端
cd frontend
npm run build

# 构建后端
cd backend
mvn clean package
```

## API文档

### 基础信息

- 基础路径：`/api`
- 数据格式：JSON

### 接口列表

#### 1. 获取所有教室及当前状态

```
GET /api/rooms
```

**响应示例：**
```json
[
  {
    "id": 1,
    "roomName": "博学楼102",
    "seatCount": 115,
    "floorNum": 1,
    "roomType": "普通教室",
    "isOccupied": true,
    "currentCourses": ["遗传学"]
  }
]
```

#### 2. 搜索教室

```
GET /api/rooms/search?keyword={keyword}
```

**参数：**
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| keyword | string | 是 | 搜索关键词 |

#### 3. 获取教室详情及课表

```
GET /api/rooms/{id}
```

**响应示例：**
```json
{
  "id": 1,
  "roomName": "博学楼102",
  "seatCount": 115,
  "floorNum": 1,
  "roomType": "普通教室",
  "currentWeek": 13,
  "weeklySchedule": {
    "1-1": [
      {
        "courseName": "遗传学",
        "teacher": "余姣君",
        "classInfo": "生科202301;生科202302",
        "studentCount": 58,
        "isBorrowed": 0,
        "borrowReason": null
      }
    ]
  }
}
```

**weeklySchedule 键格式：** `{dayOfWeek}-{period}`，如 `1-1` 表示周一第1-2节

#### 4. 查询教室指定时间状态

```
GET /api/rooms/{id}/status?week={week}&day={day}&period={period}
```

**参数：**
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| week | int | 是 | 周次 |
| day | int | 是 | 星期几（1-7） |
| period | int | 是 | 节次（1-12） |

#### 5. 获取当前时间信息

```
GET /api/time
```

**响应示例：**
```json
{
  "currentWeek": 13,
  "currentDay": 7,
  "currentPeriod": 0,
  "currentDate": "2026-06-01"
}
```

## 数据结构

### Excel文件格式

每个Excel文件代表一个教室的课表，格式如下：

| 行 | 列 | 内容 |
|----|-----|------|
| 0 | 0-8 | 学期信息、教室名、座位数 |
| 1 | 0-8 | 表头：节次、周一~周日 |
| 2 | 2-8 | 1-2节课数据（周一~周日） |
| 3 | 2-8 | 3-4节课数据 |
| 4 | 2-8 | 5-6节课数据 |
| 5 | 2-8 | 7-8节课数据 |
| 6 | 2-8 | 9-10节课数据 |
| 7 | 2-8 | 11-12节课数据 |
| 8 | 0-8 | 注释说明 |

### 课程数据格式

**普通课程：**
```
课程名/(节次)周次/教师/班级/教学班人数：XX
```

示例：
```
遗传学/(1-2节)1-10周/余姣君/生科202301;生科202302/教学班人数：58
```

**借用记录：**
```
审核状态：借用审核通过/借用原因:xxx/(节次)周次/负责人
```

示例：
```
审核状态：借用审核通过/借用原因:用于答辩-外国语学院2026届本科毕业论文答辩/(1-2节)10周/罗彬
```

### 周次格式

| 格式 | 含义 | week_type值 |
|------|------|-------------|
| 1-10周 | 每周 | 0 |
| 1-15周(单) | 单周 | 1 |
| 2-16周(双) | 双周 | 2 |

### 数据库表结构

#### room 表

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT | 主键 |
| room_name | VARCHAR(50) | 教室名 |
| seat_count | INT | 座位数 |
| floor_num | INT | 楼层（负数表示地下） |
| room_type | VARCHAR(20) | 类型：普通教室/实验室 |

#### schedule 表

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT | 主键 |
| room_id | BIGINT | 教室ID |
| course_name | VARCHAR(200) | 课程名 |
| teacher | VARCHAR(50) | 教师 |
| class_info | VARCHAR(500) | 班级信息 |
| student_count | INT | 教学班人数 |
| day_of_week | TINYINT | 星期几（1-7） |
| period_start | TINYINT | 开始节次 |
| period_end | TINYINT | 结束节次 |
| week_start | INT | 开始周次 |
| week_end | INT | 结束周次 |
| week_type | TINYINT | 周次类型（0每周/1单周/2双周） |
| is_borrowed | TINYINT | 是否借用 |
| borrow_reason | VARCHAR(500) | 借用原因 |

## 后端核心逻辑

### Excel解析流程

1. 应用启动时检查数据库是否已有数据
2. 如果没有，扫描Excel目录下所有`.xls`文件
3. 解析每个文件：
   - 提取教室名（文件名）
   - 提取座位数（Row[0][7]）
   - 提取楼层（从教室名解析）
   - 遍历Row[2-7]，解析每个时间段的课程
4. 使用正则表达式解析课程格式
5. 将数据写入数据库

### 时间判断逻辑

```java
// 当前节次判断
if (time < 800) return 0;   // 未上课
if (time < 940) return 1;   // 1-2节
if (time < 1020) return 3;  // 3-4节
if (time < 1400) return 5;  // 5-6节
if (time < 1540) return 7;  // 7-8节
if (time < 1900) return 9;  // 9-10节
return 11;                   // 11-12节
```

### 周次计算

```java
// 学期开始日期：2026-03-09
LocalDate semesterStart = LocalDate.of(2026, 3, 9);
long days = ChronoUnit.DAYS.between(semesterStart, today);
int week = (int) (days / 7) + 1;
```

## 常见问题

### Q1: Excel文件解析失败

**原因：** 文件格式不匹配或编码问题

**解决：**
- 确保Excel文件是`.xls`格式（不是`.xlsx`）
- 检查文件名是否包含教室名

### Q2: 数据库连接失败

**原因：** MySQL未启动或配置错误

**解决：**
- 确保MySQL服务已启动
- 检查`application.yml`中的数据库配置
- 确认用户名密码正确

### Q3: 前端无法访问后端API

**原因：** 跨域问题或后端未启动

**解决：**
- 确保后端已启动（端口8080）
- 检查`vite.config.js`中的代理配置
- 确认CORS配置正确

### Q4: 教室状态显示不正确

**原因：** 周次或时间判断有误

**解决：**
- 检查`SEMESTER_START`常量是否正确
- 确认当前时间在学期范围内
- 查看后端日志确认解析结果
